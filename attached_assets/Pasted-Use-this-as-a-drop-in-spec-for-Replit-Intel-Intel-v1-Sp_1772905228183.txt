Intel v1 Spec — Non-Breaking Pec Zone Proxy Layer

Goal
Add upper/mid/lower pec proxy analytics without changing the canonical 27-region muscle schema, existing seed CSV headers, muscle IDs, balance logic, recovery schema, optimizer vectors, or current frontend-facing response shapes.

Critical constraints
1. Keep `Pectorals` as the single canonical chest muscle everywhere in the existing Intel system.
2. Do NOT modify:
   - source matrix headers
   - muscles table structure or IDs
   - seed/import shape
   - existing `/muscle/day`, `/weekly_muscles`, `/coach`, `/reports`, `/optimizer` response shapes
   - balance member definitions
   - recovery tau mappings
   - exercise vectors used by current stable endpoints
3. The pec-zone layer must be additive and isolated.
4. Pec-zone outputs are proxy partitions of the existing pectoral signal, not new canonical muscles.
5. For every computed record:
   - `upper_pec_share + mid_pec_share + lower_pec_share = 1.0`
   - `upper_pec_total_dose + mid_pec_total_dose + lower_pec_total_dose = pectorals_total_dose`
   - `upper_pec_direct_dose + mid_pec_direct_dose + lower_pec_direct_dose = pectorals_direct_dose`

High-level design
Create a sidecar analytics module that takes already-computed pectoral signal and partitions it into:
- `upper_pec`
- `mid_pec`
- `lower_pec`

Use a hybrid method:
A. base zone shares by exercise archetype
B. small adjustments from existing adjacent muscle signals already present in Intel:
   - Front/Anterior Delt
   - Triceps
Optional later:
   - phase lens adjustments from V3
Not required for v1.

Implementation scope for v1
Create:
1. a new profile/config module for pec-zone base shares
2. a new service module that allocates pectoral signal into pec zones
3. new non-breaking endpoints only

No modifications to current stable endpoints unless adding optional internal calls that do not change response shape.

1. DATA MODEL
==================================================

Create a new config file:

`app/pec_zone_profiles.py`

Purpose:
Store per-exercise or per-archetype base chest-zone distributions.

Required structure:
A dict keyed by canonical exercise name, with values:

{
  "Exercise Name": {
    "upper": float,
    "mid": float,
    "lower": float,
    "archetype": str

}

Rules:
- values must sum to 1.0
- all values must be >= 0
- if an exercise is missing, fall back to archetype inference
- if archetype inference fails, use neutral default

- upper = 0.33
- mid = 0.34
- lower = 0.33

2. CORE FORMULA
==================================================

Create a new service file:

`app/pec_zones.py`

Add these functions:

A. `get_base_pec_zone_shares(exercise_name: str) -> dict[str, float]`
Behavior:
- exact-match exercise profile if present
- else infer archetype from exercise name
- else return neutral default

B. `adjust_pec_zone_shares(base, front_delt_signal, triceps_signal) -> dict[str, float]`
Behavior:
Use existing muscle signals as small directional nudges only.
Do NOT let proxies dominate the base profile.

Definitions:
- `front_delt_signal` = current exercise or set contribution for Front/Anterior Delt
- `triceps_signal` = current exercise or set contribution for Triceps
- compute:
  - `proxy_total = front_delt_signal + triceps_signal`
  - if proxy_total <= 0: return base unchanged
  - `fd_ratio = front_delt_signal / proxy_total`
  - `tri_ratio = triceps_signal / proxy_total`

Nudge logic for v1:
- front delt dominance suggests more upper pec
- triceps dominance suggests more lower pec
- balanced proxy suggests more mid pec

Recommended bounded adjustment:
- `raw_shift = fd_ratio - tri_ratio`
- `upper_delta = clamp(raw_shift * 0.10, -0.08, 0.08)`
- `lower_delta = clamp((-raw_shift) * 0.10, -0.08, 0.08)`
- `mid_delta = -0.5 * (upper_delta + lower_delta)`

Then:
- `upper = base.upper + upper_delta`
- `mid = base.mid + mid_delta`
- `lower = base.lower + lower_delta`

Apply floors:
- each zone minimum floor = 0.10

Then renormalize all three to sum to 1.0.

Important:
Adjustment magnitude must stay modest.
Base exercise profile should remain primary.
Proxy effect should only tilt the distribution, not override it.

C. `allocate_pec_zones_for_set(...)`
Input:
- exercise_name
- pectorals_total_dose
- pectorals_direct_dose
- front_delt_total_dose
- triceps_total_dose

Output:
{
  "shares": {
    "upper": ...,
    "mid": ...,
    "lower": ...
  },
  "total_dose": {
    "upper": ...,
    "mid": ...,
    "lower": ...
  },
  "direct_dose": {
    "upper": ...,
    "mid": ...,
    "lower": ...
  },
  "meta": {
    "base_profile_source": ...,
    "proxy_adjustment_applied": true/false,
    "front_delt_signal": ...,
    "triceps_signal": ...
  }
}

Dose allocation:
- `upper_total = pectorals_total_dose * upper_share`
- `mid_total = pectorals_total_dose * mid_share`
- `lower_total = pectorals_total_dose * lower_share`
- same pattern for direct dose

D. `aggregate_pec_zones(records: list[dict]) -> dict`
Sums set/day/week pec-zone outputs across records.

6. VALIDATION RULES
==================================================

Add explicit validation checks:

A. share integrity
For every allocation:
- all shares finite
- all shares >= 0
- sum(shares) == 1.0 within tolerance

B. dose conservation
- sum(zone total doses) == pectorals total dose within tolerance
- sum(zone direct doses) == pectorals direct dose within tolerance

C. non-negativity
- no negative zone doses

D. fallback safety
If pectorals total/direct dose is zero, return zeros for all zone doses and neutral shares or null shares based on your existing API style.

9. EXAMPLE PSEUDOCODE
==================================================

`app/pec_zones.py`

Pseudo only; adapt to repo style.

def clamp(x, lo, hi):
    return max(lo, min(hi, x))

def renormalize(shares):
    total = shares["upper"] + shares["mid"] + shares["lower"]
    if total <= 0:
        return {"upper": 0.33, "mid": 0.34, "lower": 0.33}
    return {k: v / total for k, v in shares.items()}

def apply_floor_and_normalize(shares, floor=0.10):
    shares = {k: max(floor, v) for k, v in shares.items()}
    return renormalize(shares)

def adjust_pec_zone_shares(base, front_delt_signal, triceps_signal):
    proxy_total = front_delt_signal + triceps_signal
    if proxy_total <= 0:
        return renormalize(base)

    fd_ratio = front_delt_signal / proxy_total
    tri_ratio = triceps_signal / proxy_total
    raw_shift = fd_ratio - tri_ratio

    upper_delta = clamp(raw_shift * 0.10, -0.08, 0.08)
    lower_delta = clamp((-raw_shift) * 0.10, -0.08, 0.08)
    mid_delta = -0.5 * (upper_delta + lower_delta)

    adjusted = {
        "upper": base["upper"] + upper_delta,
        "mid": base["mid"] + mid_delta,
        "lower": base["lower"] + lower_delta,
    }
    return apply_floor_and_normalize(adjusted)

def allocate_pec_zones_for_signal(
    exercise_name,
    pectorals_total_dose,
    pectorals_direct_dose,
    front_delt_signal,
    triceps_signal,
):
    base = get_base_pec_zone_shares(exercise_name)
    shares = adjust_pec_zone_shares(
        base,
        front_delt_signal=front_delt_signal,
        triceps_signal=triceps_signal,
    )

    total_dose = {
        "upper": pectorals_total_dose * shares["upper"],
        "mid": pectorals_total_dose * shares["mid"],
        "lower": pectorals_total_dose * shares["lower"],
    }
    direct_dose = {
        "upper": pectorals_direct_dose * shares["upper"],
        "mid": pectorals_direct_dose * shares["mid"],
        "lower": pectorals_direct_dose * shares["lower"],
    }

    return {
        "shares": shares,
        "total_dose": total_dose,
        "direct_dose": direct_dose,
    }
