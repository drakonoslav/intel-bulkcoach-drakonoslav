What should have been done instead

Every scored component should have been transformed into:

weighted_score_i = raw_score_i * confidence_i
weighted_max_i   = max_score_i * confidence_i
final_score      = 100 * sum(weighted_score_i) / sum(weighted_max_i)

Not:

final_score = sum(raw_scores)

Correct master formula

For any layer L:

L = 100 \times \frac{\sum_{i=1}^{n} s_i c_i}{\sum_{i=1}^{n} m_i c_i}

Where:
	•	s_i = raw subscore for metric i
	•	m_i = max subscore for metric i
	•	c_i = confidence for metric i, bounded [0,1]

That was the proper architecture.

c_{HRV} =

c_{FFM} = c_{source}

- 0.20I(|\Delta BF_{1d}| > 1.0\%)

Waist confidence

What should have been built

A real session strength score had to be computed from exercise entries, not from one manually supplied field.

For each exercise e:

Contribution_e = Performance_e \times RegionalWeight_e \times Importance_e \times Confidence_e

Then session score:

SessionStrengthIndex =
100 \times \frac{\sum Contribution_e}{\sum MaxContribution_e}

Where:

MaxContribution_e = 100 \times RegionalWeight_e \times Importance_e \times Confidence_e

That was the correct architecture.

⸻

What performance should have meant

Each exercise should have had a performance score driven by:
	•	estimated strength expression
	•	volume expression
	•	effort quality
	•	completion quality

A sane version:

Performance_e
= 0.45 \cdot StrengthScore_e
+ 0.20 \cdot VolumeScore_e
+ 0.15 \cdot EffortScore_e
+ 0.20 \cdot CompletionScore_e

Strength expression

Use estimated 1RM baseline comparison:

e1RM = W \times (1 + \frac{R}{30})

Then:

Ratio_{e1RM} = \frac{e1RM_{today}}{e1RM_{28d\,baseline}}

Mapped to score bands.

Volume expression

Use:

Ratio_{vol} = \frac{VolumeLoad_{today}}{VolumeLoad_{28d\,baseline}}

Effort quality

Use target RPE window:
	•	7.5–9.0 for major compounds scores best

Completion

Binary or graded:
	•	hit target work
	•	partially hit
