v2 Enhancement 1 — Exercise Geometry Detection

Instead of relying purely on exercise names, infer movement geometry.

Create a new module:

app/exercise_geometry.py

It classifies exercises into these chest geometries:

incline_press
flat_press
decline_press
dip
horizontal_adduction
vertical_adduction
pushup
neutral_press

Detection rules

Example logic:

if "incline" in name → incline_press
if "decline" in name → decline_press
if "dip" in name → dip
if "fly" or "cable fly" → horizontal_adduction
if "low to high fly" → vertical_adduction
if "push-up" → pushup
else → neutral_press

Why this matters biologically

Pec fibers run in three main orientations:

Clavicular (upper)
Sternal (mid)
Costal (lower)

Movement direction relative to torso determines which fibers shorten most.

v2 Enhancement 2 — Phase Model Signal (V3 matrices)

You already have a phase model:

Initiation
Midrange
Lockout

These contain real biomechanical information.

Use them as chest-zone hints

Rules:

Initiation dominant → upper pec bias
Midrange dominant → mid pec bias
Lockout dominant → lower pec bias

Why:
	•	upper pec contributes strongly during early shoulder flexion
	•	mid pec dominates horizontal press midrange
	•	lower pec becomes important during downward press / adduction

Implementation

From V3 matrix:

pec_init
pec_mid
pec_lock

Compute:

phase_total = pec_init + pec_mid + pec_lock

Then compute phase ratios:

init_ratio = pec_init / phase_total
mid_ratio = pec_mid / phase_total
lock_ratio = pec_lock / phase_total

Then apply bounded adjustment:

upper += init_ratio * 0.10
mid += mid_ratio * 0.10
lower += lock_ratio * 0.10

Then renormalize.

v2 Enhancement 3 — Stabilization Context

Your V5 dynamic + stabilization matrices contain useful information.

Example biological patterns

Signal	Likely Chest Bias
High anterior delt stabilization	upper pec bias
High triceps dynamic load	lower pec bias
Low stabilizer load	mid pec

Adjustment

Use ratios:

fd = front_delt_signal
tri = triceps_signal

Compute:

fd_ratio = fd / (fd + tri)
tri_ratio = tri / (fd + tri)

Adjust:

upper += fd_ratio * 0.08
lower += tri_ratio * 0.08
mid -= (upper_delta + lower_delta)/2

Clamp ±0.08.

v2 Enhancement 4 — Grip Width Proxy

Grip width dramatically shifts pec fiber recruitment.

Since Intel doesn’t explicitly store grip width, estimate it from exercise type.

Approximate mapping

wide bench press → mid pec bias
narrow press → upper + triceps bias
dips → lower pec bias
push-ups → mid bias

Implementation:

if "wide" in name → mid += 0.06
if "close grip" in name → upper += 0.04
if dip → lower += 0.08

Then renormalize.

Final v2 Algorithm

The final share calculation becomes:

shares = base_profile(exercise)
shares = apply_geometry_adjustment(shares)
shares = apply_phase_adjustment(shares)
shares = apply_proxy_adjustment(shares)

Order matters because:
	1.	exercise geometry sets baseline
	2.	phase signals refine movement mechanics
	3.	proxies refine muscular synergy
	4.	grip adjusts mechanical leverage
