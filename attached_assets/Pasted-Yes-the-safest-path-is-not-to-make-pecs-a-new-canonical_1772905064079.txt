How the pec-zone proxy should work

Use a two-stage model.

Stage 1: keep Intel’s real chest signal

For each set or day, Intel already knows:
	•	pectoral total dose
	•	pectoral direct dose
	•	fatigue / freshness of pectorals

That remains the base truth.

Stage 2: split that pectoral signal into zones

Take the pectoral dose and distribute it into upper/mid/lower shares using a hybrid of exercise profile + proxy muscles.

That gives:
	•	upper_pec_dose
	•	mid_pec_dose
	•	lower_pec_dose

with the rule:

upper + mid + lower = existing pectorals

So the chest total never changes.
You are only partitioning it.

The right modeling approach

Do not try to infer pec zones from proxies alone.

Pure proxy logic from front delts and triceps is too noisy by itself.

Instead use:

1. base exercise family profile

Examples:
	•	incline press → upper-biased
	•	flat press → mid-biased
	•	decline press / dips → lower-biased
	•	flyes / converging press → often mid-heavy with variation depending on setup

2. proxy adjustment from existing matrices

Use adjacent muscle involvement as a nudge, not the whole model.

Examples:
	•	more front-delt relative to triceps → shift slightly upward
	•	more triceps relative to front-delt → shift slightly downward
	•	more stable balanced press signature → shift toward mid pec

3. optional phase adjustment

You already have a V3 phase model.
That can later help distinguish patterns, but I would keep v1 simple.

A clean formula

For each exercise, define a base split:
	•	upper_base
	•	mid_base
	•	lower_base

that sums to 1.0.

Then apply a small adjustment from proxies.

Example concept:
	•	front delt high relative to triceps → upper += small bonus
	•	triceps high relative to front delt → lower += small bonus
	•	balanced → mid += small bonus

Then clamp all three to a minimum floor and renormalize to 1.0.

Finally:
	•	upper_pec_dose = pectoral_dose * upper_share
	•	mid_pec_dose = pectoral_dose * mid_share
	•	lower_pec_dose = pectoral_dose * lower_share

That is the safest mathematical structure.

at least not yet

They are an auxiliary analytics layer.
